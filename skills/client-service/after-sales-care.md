# Skill: 售后客户维护 (After-Sales Client Care)

## Description
系统化地进行客户售后维护，包括定期回访、持仓监控、市场波动时的主动沟通、以及客户生命周期中的重要节点关怀。

## Trigger Conditions
- 定期回访（月度/季度/年度）
- 市场大幅波动（单日涨跌超过±3%）
- 持仓基金出现重大变化（经理变更、清盘风险等）
- 客户生命周期节点（生日、投资周年、产品到期等）
- 客户主动咨询或投诉

## Workflow

### 步骤 1: 回访准备

**数据收集**:
```
□ 客户基本信息（KYC档案）
□ 当前持仓明细及市值
□ 历史交易记录
□ 上次回访记录
□ 近期市场与客户持仓表现
□ 客户近期行为（登录频率、查看报告等）
```

**分析要点**:
```
□ 组合收益率（绝对/相对）
□ 最大回撤情况
□ 配置偏离度（vs 目标配置）
□ 单品业绩归因
□ 风险暴露变化
□ 申赎行为分析
```

**制定沟通策略**:
```
□ 确定沟通目标（教育/安抚/推介/调整）
□ 预判客户可能的反应
□ 准备回应话术
□ 准备相关材料（报告、图表、产品资料）
□ 设定沟通时间和方式
```

### 步骤 2: 执行回访

**开场白**:
```
模板:
"{客户称呼}您好，我是{公司名称}的投资顾问{姓名}。

今天是{日期}，按照我们之前约定的{月度/季度}回访时间，我联系您做一个投资回顾和交流。

您现在方便吗？大概需要{15-30}分钟。"

要点:
- 自报家门
- 说明来意
- 请求许可
- 预估时间
```

**回顾环节**:
```
1. 确认客户近况
   - "最近投资方面有什么想法或困惑吗？"
   - "您的财务状况/投资目标有什么变化吗？"

2. 组合表现回顾
   - "我们先回顾一下您账户的表现..."
   - 展示收益/亏损情况（绝对值+相对排名）
   - 对比基准和同类平均
   - 分析最大回撤和恢复情况

3. 持仓分析
   - "您目前的配置是..."
   - 说明各品类表现
   - 指出表现较好/较差的单品
   - 分析原因（市场/行业/经理）

4. 目标达成情况
   - "当初设定的目标是..."
   - 对比当前进度
   - 分析偏差原因
```

**沟通与调整环节**:
```
1. 听取客户反馈
   - "您对目前的投资情况满意吗？"
   - "您有什么担心的地方吗？"
   - "您希望做哪些调整吗？"

2. 解答疑问
   - 针对客户提出的问题
   - 用通俗语言解释专业概念
   - 提供数据支撑

3. 提出建议
   - 基于市场判断给出配置建议
   - 针对客户情况提出调整方案
   - 介绍合适的产品（如有）

4. 达成共识
   - 确认是否需要调仓
   - 确定调仓方案和时间
   - 约定下次联系时间
```

**结束语**:
```
模板:
"好的{客户称呼}，今天我们主要回顾了...

总结一下，接下来的重点是...

我会{具体时间}把{材料/报告}发给您。

下次联系是{日期}，到时候我们再交流。

您还有什么问题吗？

好的，祝您投资顺利，生活愉快，再见！"
```

### 步骤 3: 回访后跟进

**记录回访内容**:
```
- 回访时间、方式、时长
- 客户情绪状态
- 客户主要关注点
- 做出的承诺和约定
- 需要后续跟进的事项
```

**发送后续材料**:
```
- 回访摘要邮件/短信
- 相关报告和分析材料
- 约定的产品资料
- 教育性内容（如适用）
```

**系统更新**:
```
- 更新客户档案
- 更新下次回访提醒
- 标记需要特别关注的事项
- 更新客户分级/标签
```

## Special Scenarios

### 场景 1: 市场大幅波动时的主动沟通

**触发条件**:
- 单日涨跌 ±3% 以上
- 连续 3 日累计涨跌 ±5% 以上
- 客户持仓最大回撤超过预警线

**沟通策略**:
```
1. 及时性原则
   - 市场在交易时间内的重大波动，收盘后2小时内联系
   - 非交易时间的重大事件，次日开盘前联系

2. 安抚为主
   - 首先确认客户的情绪状态
   - 解释波动的原因（避免过度解读）
   - 回顾历史类似情况的市场表现
   - 强调长期投资的必要性

3. 提供数据支撑
   - 展示组合相对于市场的表现（是否跑赢）
   - 说明最大回撤是否在预期范围内
   - 展示历史恢复时间

4. 给出建议
   - 不建议在市场恐慌时割肉
   - 如果有闲置资金，可考虑逢低加仓（如有定投计划）
   - 重申投资纪律

5. 约定后续
   - 告知会持续关注市场
   - 约定下次联系时间
   - 留下联系方式，有问题随时联系
```

**话术示例**:
```
"{客户称呼}您好，我是{姓名}。

今天市场出现了较大波动，{上证指数/创业板指}下跌了{X}%，我想和您做一个沟通。

首先，您现在心情怎么样？有没有特别担心？

[听取客户反馈]

我理解您的担心。这次调整主要是因为{原因简述}。

不过我想和您分享几个数据：
1. 虽然今天跌了{X}%，但您的组合跌了{Y}%，跑赢了市场{Z}个百分点
2. 您目前的最大回撤是{X}%，还在我们当初设定的{Y}%的承受范围内
3. 历史上类似的调整，平均{X}天就恢复了

而且，我们要记住，投资是长期的事。短期的波动是正常的。

我的建议是：
1. 不建议在这个时候割肉
2. 如果您还有闲置资金，可以考虑定投加仓
3. 保持平常心，该干嘛干嘛

您觉得呢？还有什么担心的吗？

好的，我会持续关注市场，有任何变化我会第一时间联系您。

您也可以随时联系我。祝您投资顺利，保持好心情！"
```

### 场景 2: 持仓基金发生重大变化

**触发条件**:
- 基金经理变更
- 基金公司重大事件（股权变更、合规问题等）
- 基金规模暴增/暴跌
- 基金投资策略发生重大调整
- 基金面临清盘或合并风险

**沟通策略**:
```
1. 及时通知
   - 第一时间联系客户，不要等客户自己发现
   - 说明发生了什么，影响是什么

2. 客观分析
   - 分析变化对基金的影响（正面/负面/中性）
   - 对比历史类似情况的后续发展
   - 评估基金经理/公司的应对能力

3. 给出建议
   - 继续持有/密切关注/考虑调出
   - 如果调出，建议替代方案
   - 设定观察期和决策点

4. 持续跟进
   - 约定观察期后的回顾时间
   - 承诺持续跟踪事件进展
   - 留下随时沟通的渠道
```

### 场景 3: 客户投诉处理

**处理原则**:
```
1. 先处理情绪，再处理问题
   - 倾听，让客户发泄
   - 同理心回应
   - 不要辩解或推卸责任

2. 明确问题
   - 澄清投诉的具体内容
   - 确认客户的期望

3. 及时响应
   - 能当场解决的当场解决
   - 不能当场解决的，给出处理时限
   - 及时反馈处理进展

4. 总结改进
   - 分析投诉原因
   - 改进工作流程
   - 预防类似投诉
```

## Commands

```bash
# 客户回访
/client-review schedule [client_id] [date]
  安排回访计划

/client-review log [client_id]
  记录回访内容

/client-review list [client_id]
  查看历史回访记录

# 客户关怀
/client-care alert [client_id]
  设置客户关怀提醒（生日、到期等）

/client-care send [client_id] [type]
  发送关怀内容（生日祝福、市场点评等）
```
